# Homara · Brain (Obsidian Vault)

Vault de conocimiento, estrategia y operaciones del proyecto **Homara**.

## Setup recomendado en Obsidian

1. **Abrir como vault**: `File → Open vault → Open folder as vault` y selecciona la carpeta `brain/`.
2. **Plugins core sugeridos** (Settings → Core plugins):
   - Backlinks · Outline · Outgoing links · Tag pane · Templates · Graph view · Page preview · Quick switcher · Search · File recovery.
3. **Plantillas**: Settings → Templates → Template folder location: `_Plantillas`.
4. **Adjuntos**: Settings → Files & Links → Default location for new attachments: `_Adjuntos`.
5. **Wikilinks**: Settings → Files & Links → Use `[[Wikilinks]]` = ON. New link format = `Shortest path`.
6. **Plugins comunitarios recomendados** (opcional):
   - **Dataview** · **Templater** · **Excalidraw** · **Kanban** · **Periodic Notes** · **Tag Wrangler** · **Iconize** · **Style Settings**.

## Convenciones rápidas

- Carpetas con prefijo numérico para orden estable (`01-…`, `02-…`).
- Carpetas con guion bajo (`_Plantillas`, `_Adjuntos`) van al inicio.
- Ficheros `_MOC-Xxx.md` son **Maps of Content** del área.
- Nombres de ficheros: `Kebab-Title-Case` en español sin acentos para evitar problemas de sync (los `aliases` en frontmatter sí pueden llevar acentos).

Ver [[00-Sistema/Convenciones-de-naming]] y [[00-Sistema/Sistema-de-tags]].
