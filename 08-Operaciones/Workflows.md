# Workflows operativos

## 1. Crear un artículo nuevo
1. Idea → [[../04-Contenido/Calendario-editorial|Calendario editorial]].
2. Brief → [[../04-Contenido/Briefs-modelo]].
3. Generación IA + revisión humana → [[../04-Contenido/Pipeline-de-IA]].
4. Carga en panel admin con productos y links afiliados protegidos.
5. [[../04-Contenido/Checklist-publicacion|Checklist]] antes de publicar.
6. Monitorización 7/30/90 días.

## 2. Crear una categoría / subcategoría
1. Validar volumen + intención ([[../03-SEO/Keywords]]).
2. Definir slug y jerarquía ([[../03-SEO/Estructura-de-categorias]]).
3. Texto SEO único.
4. Productos destacados + interlinking a artículos del cluster.
5. Añadir a sitemap y revisar `robots`.

## 3. Refrescar pieza existente
1. Detectar caída de tráfico o producto descatalogado.
2. Verificar enlaces afiliados.
3. Reescribir secciones desactualizadas.
4. Reindexar en Search Console si el cambio es relevante.

## 4. Cambio sensible en producción
1. Documentar decisión con [[../_Plantillas/Plantilla-Decision]].
2. Backup.
3. Plan de rollback escrito.
4. Deploy.
5. Verificación.
