# Pipeline de IA

Flujo para producir contenido en volumen sin perder calidad ni romper la base técnica.

## Fases

1. **Planificación** — cluster de keywords y tipo de pieza ([[Tipos-de-articulos]]).
2. **Brief** — usando [[Briefs-modelo|brief modelo]]: keyword, intención, audiencia, productos, ángulos obligatorios, longitud, internal links.
3. **Generación IA** — primer borrador siguiendo [[Estilo-editorial]].
4. **Revisión humana** — verificación factual, tono, citas, datos numéricos, enlaces afiliados.
5. **Enriquecimiento** — imágenes, schema, productos cargados, enlaces internos.
6. **Publicación** — pasa por [[Checklist-publicacion|checklist]].
7. **Post-publish** — indexación, monitoreo de posición y CTR a 7/30/90 días.

## Reglas duras
- Nunca publicar sin revisión humana.
- Nunca enlazar productos no verificados (o con afiliado caído).
- Nunca generar opinión inventada sobre experiencia de uso.

## Pendiente
- Documentar prompts maestros por tipo de pieza.
- Métricas de calidad post-publish.
